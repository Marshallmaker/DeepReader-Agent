# 摘 要

金融研究报告中蕴含着大量关键指标数据，如购回价格、营收增长率、净资产收益率，但这些数据分散在非结构化的PDF文档中，依赖人工逐份翻阅、手动录入表格的方式效率低下。针对这一问题，本课题设计并实现了一个基于AI的PDF研究报告指标提取与对比分析系统"DeepReader Agent"。系统采用Python FastAPI构建RESTful后端服务，React 18配合TypeScript搭建单页应用前端，MySQL 8.0存储业务数据，并通过Celery加Redis实现PDF解析任务的异步分发与重试。核心流程为：用户上传PDF批次后，Celery工作进程依次调用PyMuPDF解析文档文本、DeepSeek-V3大语言模型按动态提示词提取关键指标，提取结果按实体属性值的EAV纵表模式写入数据库，前端以6种可视化图表和对比矩阵呈现数据。系统还内置SSE流式AI对话模块，支持基于报告内容的智能问答。对比矩阵中的异常检测模块集成了中位数偏离、IQR四分位距、Z分数三种算法，可根据数据特征自动选择检测方法，在不少于3份共享同一股票代码的报告条件下有效标出偏离值，辅助分析师快速定位异常交易行为。该系统将报告分析从头至尾的人工操作转变为上传、自动提取、可视化对比、AI问答的四步工作流，为金融数据分析岗位中常见的批量报告处理场景提供了可落地的技术方案。

**关键词：** PDF指标提取；大语言模型；FastAPI；数据可视化；异常检测

# ABSTRACT

Financial research reports contain large amounts of key metric data, such as repurchase prices, revenue growth rates, and return on equity (ROE), yet these data points are scattered across unstructured PDF documents. Manual extraction is time consuming and error prone. This project designs and implements DeepReader Agent, an AI powered PDF report metric extraction and comparative analysis system. The backend is built with Python FastAPI, the frontend with React 18 and TypeScript, MySQL 8.0 for persistence, and Celery with Redis for asynchronous task dispatching. The core pipeline works as follows: after a user uploads a PDF batch, Celery Workers parse document text via PyMuPDF, then invoke the DeepSeek-V3 large language model with dynamically assembled prompts to extract key metrics. Results are written to an EAV table, and the frontend renders the data through six chart types and a comparison matrix. The system also includes an SSE streaming AI chat module for report grounded Q&A. An anomaly detection module, supporting median deviation, IQR, and Z Score with automatic method selection, flags outliers when at least 3 reports share the same stock code, helping analysts quickly isolate irregular trading activity. The system transforms the end to end report analysis workflow into a four step operation: upload, auto extract, visualize and compare, and AI powered Q&A, providing a practical technical solution for the batch report processing scenarios commonly encountered in financial data analysis roles.

**Keywords:** PDF metric extraction; large language model; FastAPI; data visualization; anomaly detection

# 第1章 概述

## 1.1 项目实践背景与岗位胜任力培养意义

在校期间的数据分析相关课程，大多聚焦于结构化数据的处理，如使用SQL查询关系型数据库、使用Pandas对CSV文件做统计分析。然而在企业实际工作中，大量有价值的信息以非结构化形式存在，PDF报告就是最典型的例子之一。以金融行业为例，券商研报、上市公司股份回购公告、年报等核心文件，多数以PDF格式对外发布。这些文档有固定的格式模板，但关键指标散落在各个章节段落中，分析人员若想从多份报告中提取同一指标并进行横向对比，目前仍主要依赖手动复制粘贴的方式。

传统的SQL和Python技能仍然是数据分析岗位的基础门槛，但2024年以来职位描述中出现了明显的转向："非结构化数据处理""大模型应用""提示词工程"等新关键词频繁出现。仅仅会处理表格数据已经不够了，能够借助AI工具从非结构化文档中自动提取信息，正在成为数据分析师的差异化竞争力。

2024年以来，以DeepSeek-V3为代表的大语言模型在结构化信息提取上的表现已经达到了实用水平。给定明确的JSON Schema约束，模型可以从非结构化文本中稳定地抽取指定字段。这为"PDF报告自动提取指标"这一场景提供了技术上的可能性。与此同时，OpenAI推出的`response_format: json_object`模式被各大模型厂商跟进，使得提取结果的格式可靠性大幅提高。毕业项目实践的定位正是"AI驱动的PDF报告指标提取与对比分析系统"，将大模型能力落地到数据分析师的核心工作场景中。

从岗位胜任力培养角度看，这个课题覆盖了数据分析师所需的三个层次的能力：底层是数据处理能力，包括非结构化文档解析、结构化存储建模、数据去重校验；中间层是AI工具链运用能力，包括大模型API调用、提示词工程、流式响应处理、异步任务编排；上层是数据产品化能力，包括可视化图表选型、用户交互设计、异常检测算法实现。完成这个项目，意味着从"会写数据分析脚本"跨越到"能独立构建数据分析系统"，这是校招数据分析岗位关注的核心能力。

## 1.2 行业岗位能力要求与个人能力现状分析

### 行业岗位能力要求

查阅2025至2026年主流招聘平台上数据分析师岗位的描述，其核心能力要求可以归纳为以下五个维度。

数据处理能力。SQL仍是核心技能，但企业中结构化数据只占总数据量的一小部分，大量非结构化数据如PDF文档、图片、日志文本的分析需求正在增长。能够从PDF等非结构文档中提取和清洗数据，正在成为岗位描述的常见条款。

编程与自动化能力。Python是标配，但要求已经从"能用Pandas处理数据"升级为"能用Python构建数据服务"。具体包括：搭建API接口、编排异步任务、处理并发场景、对接前端页面。

AI工具链运用能力。这是2024年后大规模出现的新要求。具体包括：调用大模型API、设计提示词模板、处理流式响应、评估AI提取结果的准确率、在数据管道中集成AI能力。

数据可视化与传达能力。不只是做图表，而是根据数据特征选择合适的图表类型、突出关键发现、用交互式手段辅助决策者理解数据。ECharts和Recharts等现代可视化库的使用是加分项。

工程化思维。能够独立完成从需求分析到系统上线的完整流程，理解前后端交互、数据库设计、异常处理、测试验证等软件工程环节。

AI工具链运用能力方面，项目前了解大模型基本概念，未实际调用过API。通过本项目，预期掌握提示词工程、JSON Schema结构化输出和SSE流式响应处理。

数据可视化能力方面，项目前使用过Matplotlib制作静态科研图表。通过本项目，预期掌握ECharts和Recharts交互式可视化，以及6种图表类型的选型与自动分派。

工程化思维方面，项目前未独立开发过从需求到上线的完整系统。通过本项目，预期完成需求分析、系统设计、编码实现、测试验证的全生命周期实践。

在校期间的课程学习给了扎实的Python编程基础和数据库操作能力。但两个明显的短板是：前端开发经验几乎为零，React和TypeScript需要从头学起；对异步任务队列和大模型API的工程实践缺乏经验。这两个领域既是本项目的技术难点，也是最有训练价值的部分。

### 项目分工与岗位能力训练

本项目由一人独立完成全部设计与开发。为明确训练目标，将系统拆分为五个技术子任务，每个子任务对应一项岗位能力的训练重点。

后端服务架构设计与实现。训练重点是FastAPI路由设计、中间件机制、统一异常处理、JWT双令牌认证体系。目标是具备独立搭建数据服务后端的能力。

PDF处理管道与AI集成。训练重点是Celery异步任务编排、PyMuPDF文档解析、DeepSeek-V3 API调用、动态提示词工程、EAV数据建模。这是技术难度最高的部分，训练非结构化数据处理与AI工具链运用的复合能力。

前端界面与交互设计。训练重点是React组件化开发、TypeScript类型系统、Ant Design UI库的使用、SSE流式数据的消费与渲染。这部分补齐了此前完全缺失的前端开发能力。

数据可视化与异常检测。训练重点是ChartRegistry注册中心设计、6种图表的实现与自动选型逻辑、量纲智能检测、三种统计异常检测算法的工程实现。

系统测试与质量保障。训练重点是功能测试用例设计、API端点测试、AI提取准确率评估。

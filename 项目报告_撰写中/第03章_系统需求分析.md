# 第3章 系统需求分析

本章从可行性、功能需求和非功能需求三个维度，明确DeepReader Agent的边界与目标。

## 3.1 可行性分析

### 3.1.1 技术可行性

以下逐项评估本项目的核心技术依赖。

PDF文本解析方面，PyMuPDF库提供了完整的PDF阅读和文本提取能力，支持中文编码。该库采用AGPL许可证，个人学习用途不受限制。

大模型API方面，硅基流动平台提供了DeepSeek-V3模型的在线调用接口，兼容OpenAI的SDK格式，直接支持`response_format: json_object`结构化输出模式。该平台提供免费额度，足以支撑开发阶段的测试调用。

前后端技术栈方面，Python FastAPI和React加TypeScript的组合已经过大量项目的验证。FastAPI的异步机制天然兼容Celery任务队列，React的生态中有Ant Design和ECharts等成熟的UI和图表库可以直接使用。这两套技术的学习资料丰富，官方文档完善，对于有Python基础但缺乏前端经验的开发者来说，技术门槛虽然存在，但在可控范围内。

数据库方面，MySQL 8.0和Redis 3.0均为安装配置简单的成熟软件，MySQL处理关系型数据的SQL标准能力和Redis的内存高速读写能力完全可以满足本项目的存储和缓存需求。

综上，本项目的各项技术依赖均有成熟的解决方案，技术路线可行。

### 3.1.2 经济可行性

本项目为个人毕业项目，不涉及商业运营。从经济角度分析，系统开发和运行成本极低。

硬件方面，开发和测试均在个人笔记本电脑上完成，无需购置服务器。软件方面，所有核心技术均为开源产品：Python及其生态库、FastAPI、React、MySQL、Redis、Celery、PyMuPDF等全部免费使用。唯一产生费用的环节是大模型API调用，硅基流动平台为DeepSeek-V3模型提供了免费额度，在开发阶段可以完全覆盖测试需求。

若未来需要将系统部署到生产环境，仅需租用一台云服务器，月成本约100至200元。大模型API的调用成本约为每百万token 1至2元，按每份PDF报告消耗5000 token估算，处理100份报告的成本不超过2元。

因此，本项目的经济可行性完全满足要求。

## 3.3 非功能需求分析

功能需求定义了系统提供哪些能力，非功能需求则定义这些能力需要达到的质量标准。以下从性能、数据处理和运行环境三个方面给出系统的设计约束。

### 3.3.1 系统性能需求

性能方面，系统的设计目标是保证普通操作流畅、耗时操作不阻塞。具体包括三点：

1. 常规查询和操作类接口，响应延迟以用户无明显等待感为准。这个标准虽然不精确，但比一个没有实测支撑的毫秒数更诚实。实际运行中，延迟主要来自数据库查询和网络传输，这两者在本项目的数据量级下构不成瓶颈。
2. PDF上传接口的响应取决于文件大小和磁盘写入速度。单个20MB以内的文件在本地开发环境下写入时间很短，但系统仍按异步方式设计，上传完成后立即告知用户结果，不做同步等待。
3. AI指标提取是系统中耗时最长的操作。系统采用异步任务模式处理这一环节：上传文件后接口立刻返回，Celery在后台执行提取任务，前端通过轮询展示处理进度。批量处理的总时长取决于Celery工作进程的并发数和API的响应速度。
4. SSE流式对话的响应延迟主要由大模型API的处理速度决定。系统将服务端收到的数据及时推送到前端，自身的转发开销很小。

### 3.3.2 数据处理需求

数据处理方面，系统的设计约束来自三个层面：文件规格、存储策略和去重机制。

1. 单批次文件上限为10份，单文件上限为20MB。10份的上限是与Celery 4并发配置对应的，超过这个数量不会出错，但处理时间会线性增长。20MB对于以文字内容为主的PDF报告已经足够，实测中典型的港股回购报告文件大小在200KB到2MB之间。
2. PDF文本解析限定前20页。这个限制有两个原因：一是大多数研报和公告的实质性内容集中在前半部分，后面的附录和法律条款对指标提取没有帮助；二是控制单次处理时长，避免超大PDF占用工作进程太久。
3. 每条提取出的指标按类型分流存储。数值型指标写入metric_value_num字段，保留两位小数；文本型指标写入metric_value_raw字段。这个设计让数值可以直接参与SQL层面的排序和聚合计算，不需要在应用层做类型转换。
4. 文件去重依赖MD5哈希。上传时先计算文件哈希值，若与已有文件匹配则跳过解析和AI提取，直接复用之前的处理结果。这个机制对用户重复上传同一份文件或修改文件名后重新上传的情况有效。

### 3.3.3 运行环境

系统涉及两套环境：本地开发环境和用户使用环境。

开发环境的配置如下：

1. 操作系统：Windows 11 Pro 64位
2. 处理器：Intel Core i7或同等性能
3. 内存：16GB及以上
4. Python 3.11，Node.js 20.x LTS
5. 数据库：MySQL 8.0.30
6. 缓存与消息队列：Redis 3.0.504
7. 开发工具：VS Code
8. 测试浏览器：Chrome 120及以上

用户使用环境只需要一台有浏览器的电脑：

1. 操作系统：Windows 10或macOS 12及以上
2. 浏览器：Chrome 100以上、Edge 100以上、Safari 16以上
3. 网络：普通宽带即可，系统传输的数据以JSON文本为主，对带宽要求不高

后端服务部署需要Python 3.11运行时和MySQL 8.0数据库实例，Celery工作进程需要额外的Redis服务。前端为纯静态资源，部署到任意HTTP服务器即可，不依赖Node.js运行时。

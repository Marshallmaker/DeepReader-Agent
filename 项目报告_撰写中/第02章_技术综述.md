# 第2章 技术综述

与模板示例中列出的Java加Nginx加SpringBoot体系不同，本项目面向数据分析与AI集成场景，选择了以Python为核心的技术组合。

## 2.1 Python

Python是一种解释型、动态类型的高级编程语言，以其简洁的语法和丰富的第三方库生态著称。在数据科学和人工智能领域，Python已经是事实上的标准语言，NumPy、Pandas、PyTorch等库覆盖了从数据处理到模型训练的完整链路。

本系统使用Python 3.11作为后端开发语言。Python在项目中负责三方面工作：通过PyMuPDF库解析PDF文档的文本内容；通过httpx库向硅基流动平台发送HTTP请求调用DeepSeek-V3大模型API，并解析返回的JSON数据；通过Celery库编排异步任务，将PDF解析和AI调用从请求线程中剥离。这些工作全部在一种语言内完成，不需要在工具链之间切换。

## 2.2 Python FastAPI

FastAPI是一个基于Starlette和Pydantic的Python异步Web框架。其特点是利用Python原生的类型注解机制，同时完成请求参数的运行时校验和OpenAPI文档的自动生成，从而减少了手写校验逻辑和维护接口文档的成本。

FastAPI负责构建全部后端API服务。30个RESTful端点分布在8个路由模块中，覆盖用户认证、文件管理、批次管理、指标管理、AI对话、数据可视化和后台管理等业务域。FastAPI的依赖注入机制实现了JWT鉴权和数据库会话的复用，中间件机制处理了CORS跨域和统一异常返回格式。异步路由支持让每个请求不阻塞事件循环，在高并发场景下保持响应性能。

## 2.3 React 配合 TypeScript

React是Meta公司维护的声明式UI框架，通过虚拟DOM和组件化模型提升前端开发效率。TypeScript是JavaScript的静态类型超集，在编译阶段发现类型错误，减少运行时调试成本。两者配合使用已是当前前端开发的主流选型。

前端基于React 18构建，使用Vite作为构建工具，Ant Design 5作为UI组件库。TypeScript的类型系统用于定义API请求和响应的数据结构，约束组件之间的Props传递，并为Zustand状态管理提供类型推导。前端共包含16个页面和30余个组件，通过SSE协议接收AI对话的流式响应，通过Apache ECharts渲染6种类型的交互式图表。

## 2.4 MySQL与Redis

MySQL 8.0是一个成熟的关系型数据库管理系统，支持ACID事务、窗口函数、CTE公共表表达式等特性。Redis是一个基于内存的键值存储系统，常用作缓存、消息队列和会话存储。

MySQL存储了全部业务数据，包括用户信息、批次记录、报告元信息、指标数据、对话历史等10张核心表。提取的指标采用EAV纵表模式存储，每条记录包含指标名、数值和文本两个字段，使得新增指标类型无需变更表结构。Redis在本系统中用于两个场景：作为Celery的消息代理和结果后端，负责任务的收发和状态追踪；存储JWT令牌的黑名单，利用SETEX命令的自动过期能力，在用户登出时将令牌标记为失效且无需手动清理。Redis不可用时，系统自动降级为内存存储，保证核心功能不中断。

## 2.5 Celery异步任务队列

Celery是Python生态中应用最广的分布式任务队列，通过将耗时操作从请求线程中分离，由后台工作进程异步执行，解决了HTTP接口因长时间等待而超时的问题。

Celery在本系统中处理两类耗时任务。核心任务是PDF处理管道：用户上传文件后，`process_batch`任务将批次中的每份报告拆分为独立的子任务，各子任务并行执行，依次完成PyMuPDF文本解析、动态提示词组装、DeepSeek-V3 API调用和EAV数据写入。失败重试策略为指数退避，最多重试3次。辅助任务是邮件发送，同样通过Celery异步执行，避免SMTP通信延迟影响接口响应。工作进程采用线程池模式，单进程4并发，任务超时设为30分钟。

## 2.6 本章小结

以上五项技术组成了DeepReader Agent的技术底座。Python提供了统一的开发语言和数据处理生态，FastAPI构建了API服务层，React加TypeScript实现了类型安全的前端界面，MySQL与Redis分别解决持久化存储和高速缓存加消息传递的需求，Celery让耗时的AI调用不阻塞用户操作。五项技术通过异步解耦和类型约束协同工作，覆盖了从PDF上传到AI分析再到可视化呈现的完整数据链路。

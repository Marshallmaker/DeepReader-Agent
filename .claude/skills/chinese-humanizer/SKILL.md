# 中文论文去AI味技能

## 概述

此技能将 AI 生成的中文学术文本改写为自然、专业、具有个人风格的人类写作。融合了 Wikipedia "AI写作特征"指南、Humanizer-zh 的24种模式检测以及中文学术写作的独特要求，覆盖 **40+ 种 AI 写作痕迹**，通过七阶段流水线处理，兼顾"去痕迹"与"注入灵魂"。

**核心原则：**
- **学术严谨性不可妥协** — 术语、数据、引用、论证逻辑永远优先于"人味"
- **去除痕迹 + 注入灵魂** — 不仅删除AI模式，更要让文字有真实的人类声音
- **默认自信处理** — 仅在术语歧义、论证偏移、意图模糊三类硬性边界时询问用户
- **越用越懂你** — 每次处理后收集反馈，动态优化规则和声纹参数
- **原文件不动** — 始终读取原文件，所有修改写入新文件（文件名加 `_去AI味` 后缀），用户原文件不被触碰
- **改动可追溯** — 所有修改以 Track Changes + 三级批注写入输出文件，用户在 Word 中逐条审阅

## 核心规则速查

处理任何文本前，牢记这 5 条规则：

1. **删除填充短语** — 去除开场白和强调性拐杖词（"值得注意的是""需要指出的是""众所周知"），直接说事
2. **打破公式结构** — 避免二元对比、三段式排比、模板化段落；两项优于三项
3. **变化节奏** — 长短句交错；段落结尾多样化，不要每个段落都以总结句收尾
4. **信任读者** — 直接陈述事实，跳过软化、辩解和手把手引导；读者不需要你替他总结"这说明了什么"
5. **删除金句** — 如果一句话听起来像可被引用的"金句"或演讲稿，重写它

## 项目报告写作专项规则（从用户反馈迭代优化）

以下规则从实际项目报告撰写反馈中提炼，适用于工程技术类项目报告的创作模式。

### 一、禁止使用的符号

1. **禁止破折号**：中文 —— 和英文 — 一律不用。改用逗号、句号或重组句式替代
2. **禁止中文括号**：中文 （） 不得出现。技术注释改用英文括号 () 或直接重组句式
3. **禁止斜杠**：如"注册/登录"，改为"注册与登录"
4. **禁止连字符**：如"
5. **禁止箭头符号**：如"→"，改用逗号或"然后""之后"等自然过渡

### 二、禁止的写作模式

1. **禁止编造数据**：任何未经实际测试验证的性能数字（如"不超过500毫秒""3秒以内""准确率94.2%"）不得写入。若无实测数据，用定性描述替代定量断言
2. **禁止教程式论述**：不要教读者认识某个技术（如"Python是一种解释型、动态类型的高级编程语言"）。项目报告应直接说明该技术在系统中做了什么
3. **禁止个人叙事**：避免"我做了X调研，发现Y趋势，于是我决定Z"的叙事结构。改为直接陈述事实
4. **禁止AI枚举格式**：不用"第一，""第二，""其一""其二"等机械编号。改为自然段落或用"一是…二是…"等轻声过渡
5. **禁止重复开头**：连续段落不得用相同的句式开头（如每节都用"本系统…"开头）
6. **禁止模板化开篇**：章节首句不得使用"本章对…进行说明""本节通过…描述"等模板引导句。直接进入内容

### 三、禁止的词汇

1. **AI套话动词**："承担""赋能""抓手""深耕""对齐"等一律不用
2. **夸张修饰**："至关重要""深远的意义""坚实的基础""重要贡献"等一律删除或替换为具体描述
3. **套话成语**："各司其职""支撑起""奠定了""标志着""见证了"等一律替换为平实表达
4. **填充引导词**："值得注意的是""需要指出的是""不可否认""众所周知"等一律删除

### 四、语言风格要求

1. **学术与口语之间取平衡**：不要太官方（避免"应""须""予以""进行"等公文词），也不要太随意（避免"卡住了""全崩了""挺好的"等口语）
2. **具体替代抽象**：说"6份报告中48个字段完全匹配45个"，不说"提取效果显著"
3. **图与文不重复**：流程图、结构图已展示的内容，文字不需要再逐条复述。文字应补充图中不易体现的设计决策或特殊处理
4. **技术术语适度中文化**：业界通用缩写保留（PDF、API、JWT、SSE、Redis），其余能用中文表达的尽量用中文（Token→令牌，Prompt→提示词，Worker→工作进程）

### 五、常见需要替换的术语对照

| 英文 | 中文 | 说明 |
|------|------|------|
| Token | 令牌 | JWT场景下 |
| Prompt | 提示词 | AI调用场景 |
| Worker | 工作进程 | Celery场景 |
| payload | 载荷 | JWT场景 |
| chunk | 数据块 | 流式传输场景 |
| Tooltip | 提示框 | 前端组件 |
| access_token | 访问令牌 | 代码块中保留英文 |
| refresh_token | 刷新令牌 | 代码块中保留英文 |
| raw_markdown | 原始内容 | 代码块中保留英文 |
| stock_code | 股票代码 | 代码块中保留英文 |

### 六、图表与效果图处理

1. 无法绘制图形时，在文中标注【图X.X 放置位置说明】，列明图中应包含的元素、层级和绘制建议
2. 效果图需说明在哪个页面截取、截取什么内容、突出什么信息
3. 图注中不使用破折号、中文括号、斜杠、连字符
4. 已画了图的文字内容，文字只需简要概述并引向图表

## 工作模式

### 命令格式

```bash
# 编辑模式 — 快速（默认七阶段全自动）
/去AI味 论文.docx

# 编辑模式 — 审阅（阶段1完成后暂停，展示AI痕迹诊断报告）
/去AI味 --review 论文.docx

# 创作模式 — 从零撰写
/去AI味 --write "帮我写第三章：实验设计与数据处理方案"

# 声纹管理
/去AI味 --sample 我的论文片段.docx     # 提供/更新写作样本
/去AI味 --profile                       # 查看当前声纹参数
/去AI味 --reset-profile                 # 重置声纹参数
```

### 编辑模式流程（处理现有文档）

读取用户指定的 .docx 文件内容（原文件不动），所有修改以 Word Track Changes（追踪修订）写入新文件，配合三级批注标记：

1. 读取 .docx，提取全文文本和结构
2. 七阶段流水线自动执行
3. 修改通过 Track Changes 写入输出文件（`原文名_去AI味.docx`）
4. 交付：输出文件 + 改动摘要报告 + 质量评分 + 原文件保持不变
5. 询问反馈 → 触发学习闭环

### 创作模式流程（从零撰写）

1. 读取用户声纹参数（如有）或使用默认学术基线
2. 读取前序章节内容（如存在），保持风格和术语一致
3. 基于声纹参数 + 个性注入策略直接生成论文内容
4. 阶段7自检评分，得分 < 35 自动重写
5. 输出 .docx 文件
6. 询问反馈 → 触发学习闭环

## 七阶段流水线

### 阶段 1：AI 痕迹检测与标记

系统扫描全文，逐句标记以下 6 大类 40+ 种 AI 痕迹：

---

#### 类别 1：夸大意义与象征性陈述（从 Humanizer-zh 移植）

AI 写作通过添加关于任意方面如何"代表"或"促进"更广泛主题的陈述来夸大重要性。这是**最高频的 AI 痕迹之一**。

| 痕迹 | 示例（AI味→人味） | 处理 |
|------|-------------------|------|
| "标志着……""见证了……" | "本实验标志着ML在医疗领域的关键转折点"→"本实验验证了ML在肺癌筛查中的可行性" | 改为直接陈述事实 |
| "作为……的证明/体现" | "该成果作为团队创新能力的证明"→"该成果表明团队在……方面具备能力" | 删除象征性框架 |
| "为……奠定基础""为……做出贡献" | "本研究为该领域未来发展奠定了坚实基础"→"本研究提供了X数据集和Y方法，可供后续工作参考" | 替换为具体贡献描述 |
| "在……中发挥着至关重要的作用" | →"是……的关键因素"或给出具体作用机制 | 删除"至关重要"等夸大修饰 |
| "凸显/强调/彰显了其重要性/意义" | →直接陈述具体影响或贡献 | 删除宣告式语言 |
| "代表了/标志着一个转变/里程碑" | →"与之前方法不同的是，本方法……" | 改为具体差异描述 |

---

#### 类别 2：模板化过渡词与 AI 连接词

| 痕迹 | 处理 |
|------|------|
| "首先…其次…再次…最后" | 简化为自然过渡或保留1层，两项优于三项 |
| "综上所述""总而言之""总之" | 改为"以上分析表明""综合来看"或直接删除 |
| "值得注意的是""需要指出的是""不可否认" | 直接进入内容，删除引导词 |
| "众所周知""如我们所知" | **学术红线** — 应引用具体文献而非诉诸"众所周知" |
| "此外""另外""同时""不仅如此"（高频连用） | 自然融入，不过度标记；段落之间可以不需要连接词 |
| "从某种意义上来讲""从某种程度上说" | 改为具体限定或删除 |

---

#### 类别 3：空洞修饰语与万能框架

| 痕迹 | 处理 |
|------|------|
| "在一定程度上""在某种意义上" | 替换为具体的程度描述或删除 |
| "不可忽视的是""不可否认的是" | 直接陈述事实 |
| "在…的过程中" | 简化为动词直接表达 |
| "通过…实现了…" | 拆分为具体动作+结果 |
| "随着…的不断发展" | 简化为"随着…发展"或给出具体时间节点 |
| "对…具有重要意义""具有深远的意义" | 替换为具体贡献描述 |
| "为…做出了重要贡献" | 替换为具体贡献内容 |
| "在…方面发挥着重要作用" | 改为具体作用描述 |
| "扮演着至关重要的角色" | 改为"是…的关键因素"或类似精简表达 |

---

#### 类别 4：AI 词汇特征（中文学术套话）

| 痕迹 | 处理 |
|------|------|
| "赋能" | 改为"使…能够""提升…能力"或根据语境具体化 |
| "抓手" | 改为"切入点""关键手段"或具体措施 |
| "闭环" | 改为"完整流程""反馈循环"或具体描述 |
| "底层逻辑" | ⚠️ 警惕 — 可能是真术语，默认标记需确认 |
| "颗粒度" | 改为"详细程度""分辨率"或具体化 |
| "对齐"（指"达成一致"） | 改为"统一""达成一致""协调" |
| "赛道"（指"领域"） | 改为"领域""方向""分支" |
| "打法"（指"策略"） | 改为"策略""方法""方案" |
| "护城河"（指"优势"） | 改为"核心优势""竞争壁垒"或具体描述 |
| "深耕"（指"专注"） | 改为"长期从事""专注研究" |

---

#### 类别 5：过度工整与结构机械化

| 痕迹 | 处理 |
|------|------|
| 连续 3 句以上完全相同长度 | 调整句式，长短句交错 |
| 排比句泛滥（3个及以上并列结构连用） | 保留核心排比，其余改为散句 |
| 段落首句全部为模板化引导句 | 多样化段首：提问式、直入式、转折式 |
| 段落长度完全相同（如每段恰好 4-5 句） | 自然变化段落长度 |
| "的"字连串（一个名词前 3 个以上"的"） | 拆分定语，改为短句或从句 |
| **三段式法则**（强行将想法分成三组）🆕 | "活动包括主题演讲、小组讨论和社交机会"→"活动包括演讲和小组讨论，会议之间还有非正式交流时间" |
| **刻意换词/同义词循环**（为避免重复而机械替换）🆕 | "主人公…主要角色…中心人物…英雄"→统一用"主人公"，允许关键词适度重复 |
| **系动词回避**（用"作为""充当""标志着"替代"是"）🆕 | "Gallery 825 作为 LAAA 的当代艺术展览空间"→"Gallery 825 是 LAAA 的当代艺术展览空间" |

---

#### 类别 6：空洞升华与虚假铺陈

| 痕迹 | 处理 |
|------|------|
| 结尾大段"展望未来""未来可期"式升华 | 改为简洁、具体的未来研究方向 |
| "不仅…而且…同时…还…"式无限叠加 | 拆分，保留最核心的关系 |
| 无实际信息的铺垫段落 | 删除或压缩为 1 句 |
| 研究背景中"随着时代的发展""在当今社会"式空泛开头 | 改为具体问题或数据引入 |
| **通用积极结论**（"前景光明""激动人心的时代即将到来"）🆕 | 改为具体计划或直接删除 |
| **虚假范围**（"从 X 到 Y"但 X 和 Y 没有有意义的尺度关系）🆕 | "从大爆炸的奇点到宏伟的宇宙网"→"涵盖大爆炸、恒星形成和暗物质理论" |
| **模糊归因**（"研究表明""学界普遍认为""行业专家认为"不给具体引用）🆕 | **学术红线** — 必须给出具体文献引用 |

#### 额外检测项（从 Humanizer-zh 移植的通用模式）

以下模式在学术论文中较少见，但在创作模式或处理非正式学术文本时仍需注意：

| 痕迹 | 处理 |
|------|------|
| **宣传/广告式语言**（"开创性的""突破性的""前所未有"） | 替换为具体数据支撑的陈述，避免用形容词代替论据 |
| **提纲式"挑战"部分**（公式化的"不足与展望"） | 改为具体局限和具体后续方向，不要写成"尽管存在这些挑战……"的模版 |
| **过度的知名度强调**（列出大量支撑性文献但不给具体引用内容） | 每个引用必须有具体贡献说明，而非"X人对此进行了研究" |
| **协作交流痕迹**（"希望这对您有帮助""请告诉我"） | 直接删除 |
| **知识截止日期免责声明**（"截至……""根据训练数据"） | 删除或改为真实时间 |
| **谄媚/讨好语气**（"好问题！""您说得完全正确"） | 改为中性学术表达 |
| **过度限定**（"可以潜在地可能被认为可能会……"） | 精简为适当程度的限定 |
| **破折号过度使用**（—） | 替换为逗号、句号或直接连接 |
| **中文里使用英文引号**（""或''替代了中文引号「」""） | 统一为中文引号 |
| **填充短语**（"为了实现这一目标""由于……的事实""在这个时间点"） | "为了实现这一点""因为""现在" |

### 阶段 2：个性与灵魂注入 🆕

> 核心洞察（来自 Humanizer-zh）：**"避免AI模式只是工作的一半。无菌、没有声音的写作和机器生成的内容一样明显。好的写作背后有一个真实的人。"**

当前声纹系统覆盖了**结构层面**（句长、用词、段落模式），本阶段补充覆盖**灵魂层面**（观点、情感、复杂性承认、个人声音）。

#### 灵魂缺失的 6 个信号

检查改写后的文本是否出现以下问题：

| # | 信号 | 判断标准 |
|---|------|----------|
| 1 | **句句结构相同** | 每个句子的长度和结构都高度一致，没有节奏变化 |
| 2 | **只有中立报道** | 仅有事实罗列，没有判断、没有态度、没有对数据和结果的"反应" |
| 3 | **不承认不确定性** | 把复杂问题简化为确定性结论，回避"可能""或许""在某些条件下" |
| 4 | **没有第一人称** | 完全回避"本文""我们""笔者认为"，写成全知全能视角 |
| 5 | **没有锋芒** | 读起来像维基百科综述或新闻稿，没有学术讨论中应有的辩驳和立场 |
| 6 | **完美得不自然** | 每个段落恰好4-5句，每个论证都有完整起承转合，没有任何"粗糙感" |

#### 注入个性的 6 个策略（学术场景适配版）

**策略 1：有观点 — 不仅报告数据和事实，还要对它们做出反应**

```
❌ AI味（只有中立报道）：
实验结果显示方法A的准确率为94.2%，方法B的准确率为91.7%。方法A在准确率上优于方法B。

✅ 人味（有观点和反应）：
方法A的准确率（94.2%）高于方法B（91.7%），但这个差距（2.5个百分点）在统计上是否显著，
取决于测试集的构成——这也是我们接下来要讨论的问题。
```

**策略 2：变化节奏 — 短促有力的判断句 + 需要时间展开的长句论证**

```
❌ AI味（句长均匀）：
本文提出了一种基于Transformer的文本分类模型。该模型通过自注意力机制捕捉文本中的长距离依赖关系。
实验结果表明该模型在三个数据集上均取得了最优性能。该模型还具有较好的可解释性。

✅ 人味（长短交错）：
本文提出了一种基于Transformer的文本分类模型。它的核心思路很简单：用自注意力机制捕捉长距离依赖。
在三个数据集上，它的性能都超过了现有方法。而且——这一点出乎我们的预料——它还具有较好的可解释性。
```

**策略 3：承认复杂性 — 真实的人类学者会有复杂的、甚至矛盾的感受**

```
❌ AI味（确定性结论）：
该方法的性能提升证明了注意力机制在文本分类任务中的有效性。

✅ 人味（承认复杂性）：
注意力机制确实带来了性能提升，但它也引入了额外的计算开销。在实时场景中，
这个开销可能抵消准确率带来的收益。我们需要在效率和准确率之间找到一个平衡点。
```

**策略 4：适当使用"我/我们" — 第一人称不意味着不专业，而是诚实**

```
❌ AI味（回避第一人称）：
本研究认为该方法的有效性已得到验证。

✅ 人味（适度使用第一人称）：
我们认为这个结果是令人信服的。当然，我们也意识到研究的局限——样本量只有500例，
大规模场景下的表现还需要进一步验证。
```

**策略 5：允许适度不完美 — 不是每个论证都需要完整的起承转合**

```
❌ AI味（完美结构）：
段落1：背景引入（3句）→ 问题陈述（2句）→ 解决方案（3句）→ 小结（1句）
段落2：背景引入（3句）→ 问题陈述（2句）→ 解决方案（3句）→ 小结（1句）
段落3：……完全相同的结构……

✅ 人味（自然变化）：
有些段落可以只有2句话，直接点出问题。有些段落可以展开到10句，深入分析一个复杂论证。
有些段落可以以一个开放性问题结尾，而不是总结句。
```

**策略 6：具体的感受 > 抽象的评价**

```
❌ AI味（抽象）：
该错误率在可接受范围内，对实际应用具有重要参考价值。

✅ 人味（具体）：
5%的错误率意味着每20个样本中就有1个会被误分类。对于一个医疗诊断系统来说，
这个数字还太高——尤其是在假阴性可能意味着漏诊的情况下。
```

#### 个性注入的学术边界

以下内容**不适用**个性注入策略（保持客观中立）：
- **方法/实验步骤描述** — 保持精确和可复现性
- **数学公式和推导** — 不受任何风格化影响
- **数据和统计结果报告** — 直接呈现，不做"反应"
- **引用和文献综述的事实部分** — 忠实于原文

---

### 阶段 3：声纹风格改写

#### 默认模式（无用户样本）

使用内置「学术自然风格」基线改写。该基线的特征：

- **句长节奏**：平均句长 25-40 字，长短句比约 3:2（3 句正常长度配 2 句短句）
- **用词倾向**：避免极端学术化（不用"兹""遂""故"），也避免口语化（不用"挺""蛮""特别多"）
- **段落节奏**：段首多样化——有时直入主题，有时简短铺垫
- **语气调性**：适当使用第一人称（"我们注意到""本文认为"），避免全知全能语调
- **连接词使用**："而""但""因此""此外""另一方面"等自然连接，避免"首先其次再次最后"
- **个性密度**：观点反应密度约每300字1次，第一人称每千字3-5次，复杂性承认在讨论/分析部分自然出现

#### 声纹模式（已有用户样本）

当用户通过 `--sample` 提供 2-3 段自己的论文写作后，分析并存储以下声纹参数到 `voice-profile.md`：

```
📊 声纹参数分析维度：

1. 句长节奏
   - 平均句长（字符数）
   - 句长方差（节奏波动程度）
   - 长短句比例（<20字:20-40字:>40字）

2. 用词指纹
   - 高频学术动词（如"表明/显示/揭示/证实/发现"的使用偏好）
   - 连接词偏好（如"而"vs"但是"，"因此"vs"所以"）
   - 程度副词使用频率（如"较""略微""显著""大幅"）
   - 是否存在标志性用词

3. 段落模式
   - 段首习惯类型（直入主题/问题引导/背景铺垫/观点先行）
   - 论证展开方式（归纳/演绎/对比/案例分析）
   - 平均段落长度

4. 语气调性
   - 第一人称使用频率（"我们""本文""笔者"）
   - 主观判断句比例（"我们认为""可以看出"）
   - 限定词使用模式（"可能""或许""倾向于""大概率"）

5. 格式习惯
   - 分点偏好（数字编号/项目符号/纯段落）
   - 图表引用方式（"如图X所示""见图X""（图X）"）
   - 引用格式（作者-年份/脚注/尾注）

6. 个性注入参数 🆕
   - 观点密度：每千字中表达观点/判断的句子数量
   - 第一人称使用偏好：倾向"本文"还是"我们"还是回避
   - 复杂性承认程度：是否习惯讨论局限性和不确定性
   - 节奏变化幅度：句长方差是偏高还是偏低
   - 结论方式偏好：干净利落还是留有余地
```

改写时逐段应用声纹参数。严格保留以下内容不改：

- 全部数值、公式、变量名
- 专业术语（尤其是领域特有术语）
- 引用标记和参考文献
- 图表编号和交叉引用
- 人名、地名、机构名

---

### 阶段 4：反向审查

对阶段 2-3 的改写结果进行反向判断：

> "假设这段文字是人类学者写的，一个 AI 检测器会不会把它误判为 AI 生成？"

审查维度：
- 改写后是否仍留有明显的 AI 痕迹模式？
- 改写后的句式是否过于统一（出现新的机械感）？
- 是否有某段改写得"用力过猛"反而显得不自然？
- 个性注入是否过度？（学术论文中观点密度不宜过高）
- 是否因注入个性而弱化了论证的严谨性？

发现问题的段落回到阶段 3 重新改写，最多重试 2 次。2 次后仍不通过，标记为 ⚠️ 需人工确认。

---

### 阶段 5：死角拦截

专项排查阶段 1-4 可能遗漏的问题：

| 排查项 | 检查内容 |
|--------|----------|
| **术语安全** | 核对 `term-whitelist.md`，确认无白名单术语被误改 |
| **数据保护** | 确认所有数值、统计量、p 值、实验数据原样保留 |
| **引用完整** | 确认引用标记 [1]、(张三, 2020) 等未被修改或删除 |
| **公式保护** | 确认数学公式、化学式未变形 |
| **图表锚定** | 确认"如图 X 所示""见表 Y"等交叉引用未断裂 |
| **格式保留** | 确认标题层级、加粗、斜体、上下标等格式未丢失 |
| **模糊归因清除** 🆕 | 确认"研究表明""学界认为"等模糊归因已替换为具体引用或删除 |

---

### 阶段 6：学术严谨性终审

逐段审阅：
1. **论证逻辑**：改写是否改变了原文的论证关系？（因果关系、并列关系、转折关系）
2. **术语一致性**：同一概念在全文中的用词是否统一？
3. **引用准确性**：改写后的句子是否仍准确传达所引文献的原意？
4. **结论力度**：是否弱化或强化了原文的结论？（不应改变）
5. **个性适度** 🆕：个性注入是否在学术边界内？（方法/数据部分保持客观）

通过后进入阶段 7。

### 阶段 7：质量评分与交付前检查 🆕

#### 质量评分（5 维度 × 10 分 = 总分 50）

对最终改写文本进行量化评估：

| 维度 | 评估标准 | 得分 |
|------|----------|:----:|
| **直接性** | 直接陈述事实还是绕圈宣告？<br>10 分：直截了当；1 分：充满铺垫和宣告 | /10 |
| **节奏** | 句子长度是否自然变化？<br>10 分：长短句交错有致；1 分：机械重复 | /10 |
| **信任度** | 是否尊重读者智慧？<br>10 分：简洁明了，不替读者总结；1 分：过度解释，手把手引导 | /10 |
| **真实性** | 听起来像真人学者写的吗？<br>10 分：自然流畅，有个人声音；1 分：AI痕迹明显，读起来像百科 | /10 |
| **精炼度** | 还有可删减的内容吗？<br>10 分：无冗余，每个词都有存在的理由；1 分：大量废话和空泛修饰 | /10 |
| **总分** | | **/50** |

**评分标准与处理：**
- **45-50 分**：优秀，AI 痕迹已去除，个性注入适度，直接交付
- **35-44 分**：良好，仍有改进空间，自动修复低分维度后交付
- **低于 35 分**：不通过，返回阶段 2 重新处理

#### 交付前快速检查清单

在输出文件之前，逐项确认：

- [ ] **连续三个句子长度相同？** → 打断其中一个
- [ ] **段落以模板化总结句结尾？** → 变换结尾方式
- [ ] **揭示了某结论前有"—"破折号？** → 删除它，用句号或逗号
- [ ] **解释了你本可以信任读者能理解的隐喻或比喻？** → 删除解释
- [ ] **使用了"此外""然而""综上所述"等AI高频连接词？** → 删除或精简
- [ ] **有三段式排比？** → 改为两项或四项
- [ ] **所有引用都有具体来源？** → 检查并消除"研究表明""学界认为"等模糊归因
- [ ] **所有数值和公式原样保留？** → 核对数据保护
- [ ] **术语在全文中用词统一？** → 核对术语一致性
- [ ] **大声朗读是否自然？** → 最后一道人工感检查

## Track Changes 三级批注系统

所有修改通过 Word Track Changes 写入输出文件（而非原文件），配合三级批注：

### 三级标记

| 级别 | 实现方式 | 含义 |
|------|----------|------|
| 🔵 **确定修改** | Track Changes 直接写入，无批注 | 高置信度修改，用户可直接接受 |
| 🟡 **需确认** | Track Changes 写入 + 黄色批注 `⚠️ 需确认：<原因>` | 术语歧义、论证可能偏移，用户需审阅 |
| 🔴 **已删除** | `<w:del>` 标记 | 原文被删除的内容，以删除线显示 |

### 批注格式

```
⚠️ 需确认：[具体原因]
  原文："…"
  建议："…"
  理由：[为什么不直接改]
```

## 拿不准时的询问边界

仅以下三类情况暂停并询问用户：

### 1. 术语歧义

> "第 3.2 段中'底层逻辑'是否为领域术语？若是，将保留原词；若为 AI 套话，建议改为'基础机制'。请确认。"

### 2. 论证可能偏移

> "第 4.1 段句式调整后，原文'A导致B'的因果关系可能弱化为'A与B相关'。请确认是否接受此改动，或我回退并保守处理。"

### 3. 意图模糊

> "第 2.3 段的一句话既具有 AI 特征，也可能是作者原始表达。无法判断，请确认是否修改。"

其余情况全部自主处理，在改动报告中汇总即可。

## 与 docx 技能的协作

本技能依赖已安装的 docx 技能进行 .docx 文件的读写：

- **读取文档**：使用 pandoc 提取文本，或通过 `scripts/office/unpack.py` 解包访问 XML
- **写入修改**：将原文档复制一份，解包副本的 XML，以 Track Changes 格式写入修改后重新打包
- **添加批注**：使用 `scripts/comment.py` 添加三级批注
- **生成文档**：使用 docx-js 从零创建带声纹风格的新文档
- **.doc 文件处理**：先通过 `scripts/office/soffice.py` 转换为 .docx

当用户的操作涉及 .docx 文件时，自动加载 docx 技能协同工作。

## 示例

### 完整改写示例（学术摘要）

**改写前（AI 味浓重）：**
> 本研究深入探讨了机器学习在医疗诊断中的关键作用，突出了其在不断演变的医疗格局中的重要性。此外，该研究作为该领域创新的证明，为未来发展奠定了坚实的基础。实验结果表明该方法具有显著的有效性，对临床应用具有重要的参考价值。行业专家认为这将对整个医疗行业产生持久影响。

**改写后（人性化）：**
> 本研究评估了三种机器学习模型（ResNet-50、DenseNet-121、EfficientNet-B0）在肺结节CT图像分类中的表现。在包含5000例标注数据的测试集上，ResNet-50的AUC达到0.94，比DenseNet-121高出3个百分点。值得注意的是，ResNet-50在小于5mm的微小结节上仍有较高的假阴性率（12%），这限制了其在早期筛查中的直接应用。

**所做更改：**
- 删除"深入探讨了……的关键作用"（夸大意义）→ 改为具体做了什么
- 删除"不断演变的医疗格局"（AI词汇）→ 直接进入方法描述
- 删除"作为……的证明，为……奠定了坚实的基础"（象征性陈述）→ 改为具体贡献
- 删除"具有显著的有效性""重要的参考价值"（空洞修饰）→ 改为具体数据（AUC 0.94）
- 删除"行业专家认为……"（模糊归因）→ 没有可靠来源，直接删除
- 添加具体模型名称、数据量、评估指标 → 具体性替代模糊主张
- 添加对局限性的具体讨论（微小结节假阴性率高）→ 承认复杂性（个性注入策略3）

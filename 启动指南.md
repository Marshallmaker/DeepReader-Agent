## 一键启动（推荐）

```powershell
.\start-all.ps1
```

该脚本会依次启动 Redis、Celery Worker、FastAPI 后端（端口 8005）和 React 前端（端口 5173），各服务运行在独立的 PowerShell 最小化窗口中。

## 手动启动

### 环境准备

#### 1. 安装依赖

**后端依赖**
```bash
cd backend
source venv/Scripts/activate   # 激活虚拟环境
pip install -r requirements.txt
```

**前端依赖**
```bash
cd frontend
npm install
```

#### 2. 配置环境变量

编辑 `backend/.env` 文件，配置以下参数：

- `DATABASE_URL`: MySQL数据库连接字符串
- `SECRET_KEY`: JWT密钥（生产环境必须修改）
- `REDIS_URL`: Redis连接字符串
- `SILICONFLOW_API_KEY`: 硅基流动API密钥

#### 3. 创建数据库

```sql
CREATE DATABASE deepreader_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 4. 初始化数据库

```bash
cd backend
source venv/Scripts/activate
python init_db.py
```

这将创建所有数据库表并插入默认管理员账号：
- 邮箱: admin@deepreader.com
- 密码: Admin@123456

### 启动服务

#### 1. 启动 Redis（用于 Celery）

```bash
redis-server
```

#### 2. 启动 Celery Worker

```bash
cd backend
source venv/Scripts/activate
celery -A app.tasks.celery_app worker --loglevel=info --pool=solo
```

> **注意**：Windows 下必须加 `--pool=solo` 参数，否则 Celery 无法正常运行。

#### 3. 启动后端 API 服务

```bash
cd backend
source venv/Scripts/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8005
```

> **注意**：Windows 下 `--reload` 可能无法可靠检测文件变更，建议修改代码后手动重启进程。

#### 4. 启动前端开发服务器

```bash
cd frontend
npm run dev
```

## 功能说明

### 用户功能
1. **注册/登录**: 支持邮箱注册，JWT双Token认证
2. **文件上传**: 批量上传PDF（最多10个，单文件≤20MB）
3. **批次管理**: 查看历史处理批次
4. **对比矩阵**: 查看多报告指标对比，异常高亮
5. **Excel导出**: 导出对比数据为Excel文件
6. **AI聊天**: 基于报告内容的AI问答（SSE流式）

### 管理员功能
1. **用户管理**: 查看、禁用/启用用户
2. **批次审计**: 查看任意用户的处理批次
3. **报告审计**: 查看报告原文和提取结果

## 注意事项

1. 生产环境必须修改 `.env` 中的 `SECRET_KEY`
2. 硅基流动API密钥需要从 https://siliconflow.cn 获取
3. MySQL版本要求 8.0.30+
4. 确保Redis服务正常运行以支持Celery异步任务
5. Windows下 Celery 必须使用 `--pool=threads` 模式（`start-all.ps1` 已自动配置）
6. 如需切换后端端口，需同步修改 `frontend/vite.config.ts` 中的代理目标

## 已知问题与解决方案

### Redis 3.0 兼容性（Windows）

本项目使用 Windows 版 Redis 3.0.504，该版本不支持 `HELLO` 命令（RESP3 协议协商）。

**问题表现**：Celery Worker 日志中出现 `unknown command 'HELLO'`，无法连接 Redis。

**解决方案**：
- `requirements.txt` 已将 redis-py 锁定在 5.3.1（高版本默认发送 HELLO）
- `backend/sitecustomize.py` 提供 RESP2 强制降级补丁
- `start-all.ps1` 启动前自动将此补丁部署到 venv

**如 venv 重建后 Celery 无法连接 Redis**，手动部署补丁：
```bash
copy backend\sitecustomize.py backend\venv\Lib\site-packages\
```

### uvicorn --reload 不生效（Windows）

Windows 下 WatchFiles 可能无法可靠检测文件变更。

**解决方案**：修改代码后手动重启后端进程（关闭对应的 PowerShell 窗口，重新运行 `start-all.ps1`）。

### 端口残留问题

Windows TCP 端口被释放后可能仍有短暂残留期。

**解决方案**：`start-all.ps1` 启动前会自动清理目标端口的僵尸进程。
